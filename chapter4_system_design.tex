\section{Load Scaling Algorithm for Mixed Fabrics}
The most significant engineering challenge was safely processing mixed loads containing different fabric types. The system implements a \textbf{Concentration-Based Load Scaling Algorithm} with strict safety priorities.

\subsection{Algorithm Rules}
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{img/c4diagram7}
    \caption{Mixed Load Safety Decision Flow}
    \label{fig:loadscaling}
\end{figure}

\begin{table}[H]
    \centering
    \begin{tabular}{|l|p{8cm}|p{5cm}|}
        \hline
        \textbf{Parameter} & \textbf{Selection Rule} & \textbf{Safety Rationale} \\
        \hline
        Temperature & Minimum value across all items & Prevents shrinking/damage to delicate fabrics \\
        \hline
        Spin Time & Minimum value across all items & Prevents stretching delicate fabrics \\
        \hline
        Water Level & Calculated based on load size, capped at 60L & Prevents overflow, ensures cleaning \\
        \hline
        Mechanical Action & Gentlest action required by any item & Protects most sensitive garment \\
        \hline
        Detergent & Calculated for heaviest soil level & Ensures cleaning without excess \\
        \hline
    \end{tabular}
    \caption{Parameter Selection Rules for Mixed Loads}
    \label{tab:mixed_rules}
\end{table}

\subsection{Code Implementation}
\begin{lstlisting}[caption=Python Implementation of Load Scaling Algorithm, label=lst:loadscale, language=Python]
def combine_washing_params(logics):
    # Protect delicate clothes by identifying the safest minimum temperature
    safe_temperature = min(item.temperature_setting for item in logics)
    
    # Protect delicate clothes by identifying the lowest safe spin time
    safe_spin_time = min(item.spin_time for item in logics)
    
    # Calculate enough water for the clothes, capped strictly at 60 Liters
    calculated_water = calculate_water_based_on_items(logics)
    water_amount = min(calculated_water, 60)
    
    # Select gentlest mechanical action required
    action_priority = {"Gentle": 1, "Normal": 2, "Heavy Duty": 3}
    gentlest_action = min(logics, key=lambda x: action_priority[x.mechanical_action])
    
    # Return the final, combined safe instructions
    return {
        "temperature_setting": safe_temperature,
        "spin_time": safe_spin_time,
        "water_level": water_amount,
        "mechanical_action": gentlest_action.mechanical_action,
        "detergent_amount": calculate_detergent_for_heaviest_soil(logics)
    }
\end{lstlisting}

\section{Feedback Loop Design}
The system incorporates a Human-in-the-Loop (HITL) feedback mechanism for continuous improvement.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.9\textwidth]{img/c4diagram8}
    \caption{Self-Learning Feedback Mechanism}
    \label{fig:feedback}
\end{figure}

\textbf{Process:}
\begin{enumerate}
    \item AI makes prediction with confidence score
    \item User reviews result
    \item If incorrect, user submits correction to \texttt{/feedback} endpoint
    \item Correction stored in database
    \item On retry, system prompts AI with: ``IMPORTANT: 'Cotton' was INCORRECT! Look more carefully...''
    \item AI self-corrects and provides improved accuracy
\end{enumerate}

\section{Rate Limiting and Failover Design}
To ensure system reliability and prevent crashes from request bursts:

\begin{figure}[H]
    \centering
    \includegraphics[width=0.9\textwidth]{img/c4diagram9}
    \caption{Request Management Architecture}
    \label{fig:ratelimit}
\end{figure}

\textbf{Implementation Features:}
\begin{itemize}
    \item \textbf{Rate Limiting:} Maximum requests per minute per user
    \item \textbf{Automatic Failover:} If primary \texttt{gemini-1.5-flash} model is overwhelmed, system gracefully waits or switches to backup model
    \item \textbf{Queue Management:} Requests are queued during high traffic
    \item \textbf{Timeout Handling:} Failed requests are retried with exponential backoff
\end{itemize}
